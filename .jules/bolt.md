## 2024-05-24 - [Avoid full directory context upload for Docker builds]
**Learning:** Sending the full directory context during a `docker build` can be a huge performance bottleneck. If a script creates a Dockerfile dynamically and uses `COPY` or `ADD` from the host context, Docker needs that full context upload. However, if the Dockerfile doesn't copy local files, it's better to bypass this context upload completely by piping the Dockerfile to `docker build` via stdin (using `docker build - < Dockerfile`).
**Action:** Always check if a Dockerfile uses host context via `COPY` or `ADD`. If it doesn't, pipe the Dockerfile directly to stdin to save a significant amount of build time, especially in large directories.
## 2025-03-27 - Don't reimplement CLI internal auth logic for performance
**Learning:** Attempting to optimize a CLI startup check (like `claude -p "ok"`) by checking for environment variables or local credentials files (like `.credentials.json`) is a brittle micro-optimization of a cold path. The `claude` CLI handles its own authentication state and may store credentials in various locations (e.g., OS keychains) not just flat files. Guessing these mechanisms introduces severe functional regressions for a negligible performance benefit, as this check only runs once at startup.
**Action:** Never bypass a tool's built-in state checking commands (like `claude -p "ok"`) in favor of manual file/environment variable checks to save a fraction of a second on a one-time cold path. Correctness and robustness are more important than micro-optimizing initialization.
